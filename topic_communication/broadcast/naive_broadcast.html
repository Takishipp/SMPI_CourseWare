Augment the bcast_skeleton.c program with a broadcast implementation named naive_bcast (first command-line argument). This implementation simply has the root of the broadcast (profess with rank 0) loop over all other processes and perform a single point-to-point MPI_Send to each process (which performs an MPI_Recv). In other words, the processes are logically organized as a mono-directional star. The second command-line argument (chunk size) is ignored.
